## 1) Product goals
Build a modern finances management app that supports:
- Expense & income tracking (manual + import)
- Budgeting (envelopes/categories, monthly/weekly)
- Accounts (cash, bank, card, savings), transfers, reconciliations
- Bills & recurring payments
- Savings goals (targets, progress, automation suggestions)
- Debt tracking and payoff strategies (snowball/avalanche)
- Investments tracking (basic portfolio view)
- Net worth dashboard
- Reports (category trends, merchant insights, cashflow)
- Smart search and filters (by category, date, merchant, tag)
- Attachments for transactions (receipts)
- Multi-currency support
- Export/import (CSV, PDF summaries)
- Offline-first (works without internet, sync later)
- Secure by design (encryption, PIN/biometric lock, backups)

## 2) Special module: To-Buy / Apartment & Kid Planning List
Add a dedicated module that behaves like a “shopping + planning” system:
- Two list types:
  A) TODO list (tasks) 
  B) TO-BUY list (planned purchases)
- To-Buy items must support:
  - name, category (furniture, kitchen, baby, safety, etc.)
  - priority (low/med/high)
  - status (idea, researching, planned, ordered, delivered, installed, returned)
  - quantity
  - estimated cost and actual cost
  - preferred store / link
  - target date (when needed)
  - notes + photos
  - tags (e.g., “kid”, “urgent”, “apartment”)
  - warranty info (optional)
  - split payment / who pays (optional)
- Budget integration:
  - each To-Buy item can reserve budget (“planned spend”) even before purchase
  - ability to convert an item into a real expense transaction when bought
  - show “planned vs actual” for furnishing and kid-prep projects
- Project view:
  - “New Apartment Setup” project with rooms (bedroom, kitchen, living room…)
  - “Kid Arrival” project with checklists (essentials, medical, safety)
  - progress % per project, total planned cost, remaining cost
- Reminders:
  - notifications before target date, and for overdue items
- Optional: shared mode later (collab list), but MVP can be single-user
